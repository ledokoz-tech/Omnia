# Security Policy

## Supported Versions

We take security seriously and actively maintain security updates for the following versions of Omnia:

| Version | Supported          |
| ------- | ------------------ |
| 0.2.x   | :white_check_mark: Security updates |
| 0.1.x   | :white_check_mark: Security updates |
| < 0.1   | :x: Not supported  |

## Reporting a Vulnerability

If you discover a security vulnerability in Omnia, please help us by reporting it responsibly.

### How to Report

**Please do NOT report security vulnerabilities through public GitHub issues.**

Instead, please report security vulnerabilities by emailing:
- **Email**: security@ledokoz.tech
- **Subject**: `[SECURITY] Vulnerability Report - Omnia`

### What to Include

When reporting a security vulnerability, please include:

1. **Description**: A clear description of the vulnerability
2. **Impact**: What an attacker could achieve by exploiting this vulnerability
3. **Steps to Reproduce**: Detailed steps to reproduce the issue
4. **Affected Versions**: Which versions of Omnia are affected
5. **Environment**: Your testing environment (OS, Rust version, etc.)
6. **Proof of Concept**: If possible, include a proof of concept
7. **Your Contact Information**: How we can reach you for follow-up questions

### Our Response Process

1. **Acknowledgment**: We will acknowledge receipt of your report within 48 hours
2. **Investigation**: We will investigate the reported vulnerability
3. **Updates**: We will provide regular updates on our progress (at least weekly)
4. **Fix Development**: Once confirmed, we will work on developing a fix
5. **Disclosure**: After the fix is ready, we will coordinate disclosure with you

### Responsible Disclosure

We kindly ask that you:

- Give us reasonable time to fix the vulnerability before public disclosure
- Avoid accessing or modifying user data
- Avoid disrupting our services
- Do not perform DoS attacks or spam our systems

### Recognition

We appreciate security researchers who help keep our users safe. With your permission, we will acknowledge your contribution in our security advisory and changelog.

### Security Updates

Security updates will be released as patch versions with the following process:

1. **Patch Release**: A security patch will be released as soon as possible
2. **Security Advisory**: A detailed security advisory will be published on GitHub
3. **Changelog Update**: The fix will be documented in our changelog
4. **Backporting**: Critical fixes may be backported to supported versions

### Contact

For questions about this security policy, please contact:
- **Email**: security@ledokoz.tech
- **Response Time**: We aim to respond within 48 hours
